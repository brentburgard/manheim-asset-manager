A self-contained, zero-dependency HTML application for managing reconditioning shop assets across all Manheim locations. Track preventative maintenance schedules, monitor PM health status, and manage equipment across Mechanic, Body & Paint, and Detail shops — all from a single file.

🚀 Quick Start

No server, no install. Just open the file:

git clone https://github.com/manheim-cox/recon-shop-asset-tracker.git
cd recon-shop-asset-tracker
open recon_shop_tracker.html    # macOS
start recon_shop_tracker.html   # Windows

✨ Features

  • Network Dashboard — Red/yellow/green PM health across all 76 Manheim locations
  • Location Management — Per-location asset CRUD with shop filtering (Mechanic / Body & Paint / Detail)
  • PM Tracking — Configurable intervals (30/60/90/180/365 days) with overdue alerts
  • 102 Equipment Types — Categorized by shop with MECH, B&P, and DETL prefixes
  • Offline / Local Storage — All data persisted in localStorage, no backend required
  • Manheim Branding — Official Cox Automotive brand colors and embedded logo

🏗 Architecture

recon_shop_tracker.html
├── <style>         Manheim brand tokens, layout, component styles
├── <header>        Sticky nav — Manheim by Cox Automotive logo
├── <nav.sub-nav>   Dashboard / Location Management tabs
├── #view-dashboard  Network stats grid + location health table
├── #view-locations  Location list → detail panel
│   ├── Add Asset form (collapsible)
│   └── Assets table (sortable, edit/delete)
└── <script>        Data layer, PM logic, localStorage persistence

📋 PM Health Logic

  • 🔴 Overdue — Next PM date has passed
  • 🟡 Due Soon — Next PM due within 30 days
  • 🟢 Current — More than 30 days remaining
  • ⚪ No Data — No PM date recorded

📦 Data Seeded

Pre-loaded with real data from Manheim Atlanta (4 assets) and MARS (1 asset). All 76 locations are available to populate via the Location Management view.
